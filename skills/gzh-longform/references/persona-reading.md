# 人设读取规范

skill 运行时从**目的仓库**取人设,不写死任何固定人设。

## 读什么

目的仓库根目录的 `AGENTS.md` / `CLAUDE.md`(若 `CLAUDE.md` 是 `AGENTS.md` 的软链,读一份即可)。两者都没有 → 停下,问用户给人设或换仓库。

## 提取字段

| 字段 | 说明 |
|---|---|
| 人设口吻 | 冷静陈述者 / 教授讲座感 / 对话感等 |
| 目标人群 | 写给谁(如非技术小白 / 产品经理) |
| 定位 | 工具/内容方向(如"专业 AI 工具平民化翻译官") |
| 禁用词 | 不能用的词(低俗口语、空洞营销词等) |
| 创作规范 | 字数/结构/排版约定 |
| 工具保持专业 | 是否要求不降级消费级工具 |

## 提取方式

整篇读 `AGENTS.md`/`CLAUDE.md`,agent 自行判断提取上述字段。人设可能散落多处(开头定位段、中间"文案口吻"段、禁用词段),需通读整合,不要只看一段。

## 提取规则

- **全部字段读到** → 直接用,进入研究简报(第 3 步)的人设摘要。
- **关键字段缺失**(人设口吻或目标人群缺) → 停下,把已提取字段列给用户,问"补全或确认"。
- **完全读不到**(目的仓库无 AGENTS.md/CLAUDE.md) → 停下,问用户给人设或换仓库。
- **不臆测**:没读到的字段不编,留给用户补。

## 兜底

提取结果进入研究简报的"人设摘要(3-5 行)",由用户在**硬闸门**确认。这是人设正确性的最后一道保险——agent 判断错了,用户在此纠正。
