# 占位格式规范

article.md 里的配图占位统一格式:

```
> 🖼️ 【配图N · 尺寸 · 来源】描述
```

## 字段

| 字段 | 说明 | 示例 |
|---|---|---|
| 配图N | 序号,封面用「封面」,正文用数字 | `封面` / `配图①` |
| 尺寸 | 宽×高 或 比例 | `900×383` / `1080×自由` |
| 来源 | `生成` / `自供` | 生成=AI 出图;自供=用户提供 |
| 描述 | 画面内容(生成图必填;自供图即路径) | 画面元素、色调、重点 |

## 示例

```
> 🖼️ 【封面 · 900×383 · 生成】左右两邮箱并排,左人类邮箱堆满未读灰暗,右 AI 邮箱干净明亮,中间「隔离」分隔线。

> 🖼️ 【配图① · 1080×自由 · 生成】收件箱界面,500+ 未读 newsletter,红点密布,灰冷压抑。

> 🖼️ 【配图② · 1080×自由 · 自供】images/inbox.png
```

## 解析规则

- 必须在 `>` 引用块内,以 `🖼️` 开头
- 【】内三段用 `·` 分隔,顺序固定:序号 · 尺寸 · 来源
- **来源可缺省**,默认 `生成`(自供图必须给路径)
- 自供图:描述字段即图片路径(`images/xxx.png`)

## 草稿/定版分态

- **草稿态**:占位块保留,生成/自供图插入占位块下方,描述并存(便于改图对账)
- **定版态**:删除所有 `🖼️` 占位块,只留 `![描述](path)` 图片

## 与 gzh-longform 衔接

gzh-longform 第 4 步生成正文时,按此格式插占位。配图 skill 读取这些占位出图,不自动衔接(用户手动跑)。
